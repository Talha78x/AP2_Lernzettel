# Pseudocode

**Pseudocode** ist eine informelle, textbasierte Beschreibung eines [[Algorithmen und Kontrollstrukturen|Algorithmus]]. Er sieht aus wie eine echte Programmiersprache, folgt aber keinen strengen syntaktischen Regeln (wie Semikolons am Zeilenende), sondern soll für jeden Menschen – unabhängig von der bevorzugten [[Programmiersprachen|Programmiersprache]] – sofort verständlich sein.

In IHK-Prüfungen (sowohl für Anwendungsentwickler als auch für FISIs!) ist Pseudocode enorm wichtig. Prüfer nutzen ihn, um das reine logische Verständnis für ein Problem abzufragen, ohne dass sich der Prüfling in der Syntax von Java, Python oder C# verheddert.

**Typische Elemente im Pseudocode:**
- **Zuweisungen:** `setze Zaehler auf 0` oder `Zaehler = 0`
- **Ein-/Ausgabe:** `lese Eingabe`, `schreibe "Fehler"`
- **Bedingungen:** `WENN Alter >= 18 DANN ... SONST ...`
- **Schleifen:** `SOLANGE Datei nicht zu Ende WIEDERHOLE ...`

Für die Prüfung gilt: Auch wenn Pseudocode keine feste Syntax hat, muss er logisch korrekt, eingerückt (für die Lesbarkeit) und eindeutig sein. Variablen sollten im Idealfall am Anfang kurz deklariert oder sprechend benannt werden (z. B. `DurchschnittsTemperatur`).
