# UML Diagramme

## Definition
UML ist eine standardisierte Modellierungssprache zur Darstellung von Struktur und Verhalten eines Systems.

## Warum ist das so?
Modelle reduzieren Missverständnisse, bevor Code entsteht.

## Zusammenspiel
- Use-Case aus Fachsicht.
- Klassen-/Sequenzdiagramme für technische Umsetzung.
- Aktivitätsdiagramme passen zu [[Algorithmen und Kontrollstrukturen]].

## Diagrammtypen (prüfungsrelevant)
| Typ | Zeigt | Typische Prüfungsaufgabe |
|---|---|---|
| Use-Case | Akteure + Anwendungsfälle | Akteure/Anwendungsfälle ergänzen |
| Klassendiagramm | Klassen/Beziehungen | Assoziation vs. Vererbung |
| Sequenzdiagramm | Nachrichtenreihenfolge | API-Ablauf darstellen |
| Aktivitätsdiagramm | Prozessfluss | Verzweigungen/Schleifen modellieren |

## Beispielaufgabe
**Frage:** Wann Sequenzdiagramm statt Klassendiagramm?

**Lösung:** Bei zeitlicher Interaktion (wer ruft wen wann auf) -> Sequenzdiagramm.
