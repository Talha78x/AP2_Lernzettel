# Mock-Ups

## Definition
Mock-Ups sind visuelle Entwürfe von Oberflächen (Layout, Navigation, Interaktion), meist ohne vollständige Funktion.

## Warum ist das so?
Frühes Visualisieren spart Kosten: Fehler im Design sind vor Umsetzung viel günstiger korrigierbar.

## Zusammenspiel
- Input für [[Bildschirmausgabemasken]].
- Ableitung von User Stories und späteren Tests.

## Eigene Worte
„Mock-Ups sind die Blaupause fürs UI, bevor gebaut wird.“

## Beispielaufgabe
**Frage:** Warum zuerst Wireframe, dann Mock-Up?

**Lösung:**
Wireframe klärt Struktur/Funktionsfluss, Mock-Up konkretisiert Look&Feel. So trennt man Fachlogik von Designfragen.
