# Bildschirmausgabemasken

**Bildschirmausgabemasken** (oft auch als GUI – Graphical User Interface oder UI bezeichnet) sind die grafischen Schnittstellen, über die ein Anwender mit einer Software oder Datenbank interagiert. In Prüfungsszenarien geht es oft darum, aus einem Textentwurf oder einem Datenmodell eine sinnvolle Eingabe- oder Ausgabemaske zu skizzieren.

Gute Bildschirmausgabemasken orientieren sich stark an den Regeln der [[Softwareergonomie Barrierefreiheit]]:
- **Klare Strukturierung:** Logisch zusammengehörende Felder (z.B. Name, Straße, Ort) werden visuell gruppiert.
- **Eindeutige Beschriftungen (Labels):** Der Nutzer muss sofort erkennen, was in ein Feld eingetragen werden soll.
- **Pflichtfelder:** Müssen deutlich gekennzeichnet sein (z.B. mit einem Sternchen `*`).
- **Feedback:** Die Maske muss dem Nutzer klare Rückmeldungen geben (z.B. "Daten erfolgreich gespeichert" oder Fehlermeldungen direkt am falschen Eingabefeld).

In der Entwicklungsphase werden Masken oft zuerst grob als Wireframe skizziert und später als fertige [[Mock-Ups]] detailliert ausgearbeitet. Für Systemintegratoren ist das Thema wichtig, wenn es um das Design von internen Dashboards, IT-Service-Portalen oder [[Ticketsysteme|Ticketsystem-Masken]] geht.
