# Vorgehensmodelle (Wasserfall, V-Modell, Spiralmodell, Prototyping, Scrum, Kanban ...)

Vorgehensmodelle legen fest, wie ein Softwareentwicklungs- oder IT-Projekt strukturiert und durchgeführt wird.

**Klassische (sequenzielle) Modelle:**
- **Wasserfallmodell:** Die Phasen (Analyse → Design → Implementierung → Test → Wartung) laufen streng nacheinander ab. Eine Phase muss abgeschlossen sein, bevor die nächste beginnt. *Vorteil:* Einfach zu planen. *Nachteil:* Sehr unflexibel; Änderungen in späten Phasen sind extrem teuer.
- **V-Modell:** Erweiterung des Wasserfalls. Jeder Entwicklungsphase auf der linken Seite des "V" wird eine Testphase auf der rechten Seite direkt gegenübergestellt (z.B. Unit-Design → Unit-Test). Stärker auf Qualitätssicherung (siehe [[Testvarianten Unit-Tests, Integrationstests, Systemtests....]]) ausgerichtet.
- **Spiralmodell:** Iterativer Ansatz. Das Projekt durchläuft in immer größeren Spiralen die Phasen Planung → Risikoanalyse → Entwicklung → Evaluation. Risikomanagement steht im Vordergrund.
- **Prototyping:** Früh wird ein funktionsfähiger (aber unvollständiger) Prototyp erstellt, damit der Kunde frühzeitig Feedback geben kann. Vermeidet teure Fehlentwicklungen.

**Agile Modelle:**
Diese Modelle sind iterativ (kurze Entwicklungszyklen) und flexibel auf Veränderungen ausgerichtet (detaillierter in [[Scrum, Agile Methoden, ISO, ITIL]]).
